# JC CLI Documentation

This directory contains all documentation for the JC CLI tool.

## Documentation Structure

### 📚 User Documentation (`user/`)
Documentation for end users of the JC CLI tool:
- **README.md** - Main project overview and getting started guide
- **START_HERE.md** - Quick onboarding guide for new users
- **QUICK_START.md** - Fast-track setup and usage guide
- **CLI_README.md** - Comprehensive CLI command reference
- **CHEATSHEET.md** - Quick reference for common commands
- **QUICK_COMMANDS.txt** - Formatted quick command reference

### 🔧 Development Documentation (`development/`)
Documentation for developers working on the JC CLI codebase:
- **ARCHITECTURE.md** - System architecture and design decisions
- **TESTING.md** - Testing strategy, guidelines, and best practices
- **REFACTORING.md** - Refactoring notes and technical debt
- **REFACTOR_SUMMARY.md** - Summary of completed refactoring work

### ✨ Feature Documentation (`features/`)
Detailed documentation for specific features:
- **EXPORT_FEATURE_SUMMARY.md** - Confluence export feature details

### 📋 Examples (`examples/`)
Reserved for example outputs and exports from the tool.

## Quick Links

- **New Users**: Start with [`user/START_HERE.md`](user/START_HERE.md)
- **Developers**: See [`development/ARCHITECTURE.md`](development/ARCHITECTURE.md) and [`development/TESTING.md`](development/TESTING.md)
- **Quick Reference**: Check [`user/CHEATSHEET.md`](user/CHEATSHEET.md)

## Contributing

When adding new documentation:
1. Place user-facing docs in `user/`
2. Place developer/technical docs in `development/`
3. Place feature-specific docs in `features/`
4. Place examples and exports in `examples/`
5. Update this README with links to new documentation
